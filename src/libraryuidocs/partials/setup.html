Setup and Customization

Once download it is advisable to first install/setup without changing anything (get your staging folder) and then customize(name changes)

Setup and Installation

  1. Do all your command line installation in that 'build' folder. You can also install packages globally from within that folder using -g
  2. Install Node js globally
  3. In general now a days NPM js comes with Node installation, but you can also check with the following link, NPM Check
    if not installed then follow instruction in Install NPM
  4. Install Grunt package
  5. Install Ruby package
  6. Install Sass
  7. Now do a ‘npm install’ again to load all your require npm locally
  8. Now run the command 'grunt build' to build the framework. This will create a staging folder in your project directory. Inside the the staging folder you will find the 'index.html' file.
    If you want a continuous watch in your build command, then just run command 'grunt'
  9. Now you are all set, open browser and open the index.html file under staging folder


Customization

  1. Rename or place both the "build" and "src" folders in the custom/client name folder and then at the top main custom/client folder level find and replace the following words
    • - classnameui with their desired name e.g. aigui
    • - classnamejs with their desired name e.g. aigjs
    • - companyname with their desired name e.g. aig
    • - MyCompany with their desired name e.g. AIG Corp.
  2. Once completed then run ‘grunt build’ command

Adding CSS and HTML

When you need to add a new element or module, then you need to create both the scss file under CSS folder and the html file under templates folder. Also you need to add the snippets portion

Step One: Adding the new scss file

  1. Create a new file with .scss extension under appropriate elements or modules folder under css/****
  2. Open mainui.scss fie and add the new file name accordingly

Step Two: Adding the new html file

  1. Create a new file with .html extension under appropriate elements or modules folder under templates/***
  2. Open one of the starter-template.html found in the elements or modules folder
  3. Copy the whole content of the starter-template.html and paste it in the new file
  4. In line 2, change the id="yourid" to the id name you want to call it, remember it is the unique ID which will become the sub-menu list
  5. Change the ‘main title’ and the ‘sub title’ to your elements or modules name
  6. It is better to keep the file name, ID name and the main tile name same

Step Three: Adding the newly created file to the sub menu list

  1. Under ‘libraryuidocs/partials/’ folder you will find corresponding html files
  2. Add the newly created file name as an 'li' tag with the ID you provided.

Step Four: Adding it to the build

  1. Open the partials.json file under the ‘build’ folder
  2. Update the newly created elements or modules which you have just created within appropriate place
  3. Copy the whole content of the starter-template.html and paste it in the new file
    e.g. if it is a module then
    "templates/modules/newfile.html": "#inserted-modules",
    if it is an element then
    "templates/elements/newfile.html": "#inserted-elements",

Step Four: Adding the snippets

  1. Create a new file with .html extension under templates/snippets and under approprieate folder section to match, e.g element/modules
  2. the name of the file should be saved as "filename_filename_0_template.html" e.g. if creating for a logo.html then the snippet should be named as "logo_logo_0_template.html
  3. Paste the code portion only and save it
  4. You should be able to see the code

Adding JavaScript

The primary purpose of this library is primarily an organization of writing CSS and HTML and is advised to hand it over to development environment to use their JavaScript.
But you can also write your own JavaScript but adding into AMD module setup as it has requirejs configuration.

Step One: Adding JS file

  1. Add the new JS file under js/modules/
  2. Create and use AMD wrapper as shown in some current JS files

Step Two: Adding JS to build

  1. Open paths.json file under ‘build’ folder
  2. Add the newly created JS file appropriately
